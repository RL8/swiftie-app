üìù Technical Implementation Details

Context API Pattern Changes

Before (Version 1):

// Direct context access - incompatible with Svelte 5 runes mode
const music = getContext<MusicContext>('music');

// Or incorrect function invocation
const music = getContext<() => MusicContext>('music')();

After (Version 2):

// Store the context accessor function
const getMusicContext = getContext<() => MusicContext>('music');

// Option 1: Use it directly in derived values
let derived = $derived(getMusicContext().someProperty);

// Option 2: Initialize in onMount
let music: MusicContext;
onMount(() => {
  music = getMusicContext();
});

Implementation Notes

  • In Svelte 5's runes mode, getContext returns a function that must be invoked to access the context value
  • The pattern fix was applied consistently across all components using context API
  • Some components required additional updates to state management ($state variables)
  • The fixes maintain full backward compatibility with existing functionality
